** [.tag.is-warning]#Chitra# Fix image scale when output resolution changes. Image scale was not changed when the canvas resolution changes. For example the following example draws only in 25% of the canvas size because paper size almost increased to 4 times when the default resolution changed to 300 from 72 (https://github.com/aravindavk/chitra-d/commit/c001835c92f7f1a5af6efa4640fbd95aaad19353[link]).
+
[source,d]
----
auto ctx = new Chitra(600);
ctx.image("img_600.png", 0, 0);
ctx.saveAs("output.png", resolution: 300);
----
With this fix the above will render 600px image to full paper width. Works with any resolution setting.

* *_JUN 12, 2026 . Friday_*
** [.tag.is-warning]#Dataframes(D)# Added `shift` feature to shift the column values as needed by giving the required number of jump value. Negative jump value will shift the column values in reverse order (https://github.com/aravindavk/dataframes-d/commit/7aeec5734056e836831470e53cadc40ceca3090f[link]).
+
[source,d]
----
auto c1 = Column!int([10, 20, 30, 40, 50]);
writeln(c1.shift(1)); // Shifts index by 1 => [0, 10, 20, 30, 40]
writeln(c1.shift(-1)); // Reverse Shifts index by 1 => [20, 30, 40, 50, 0]
----
+
Sets the default value of respective type (Ex: `nan` for double, `""` for string etc) when shifted.
+
To find the average of the last two values:
+
[source,d]
----
auto c1 = Column!int([10, 20, 30, 40, 50]);
auto avgCol = (c1 + c1.shift(1)) / 2;
writeln(avgCol);
----

----

colorScale(255, 255);
fill(0, 0, 255, 127);  // Blue with Opacity 50%
----
