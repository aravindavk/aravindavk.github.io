Effective Gluster Monitoring using Events APIs

Sep 26, 2016
2 minutes read
gluster glusterfsblog

Last week I got opportunity to present about Gluster Events APIs in `Gluster meetup Bangalore <http://www.meetup.com/glusterfs-India/events/233515975>`__.

Events APIs will be available with Gluster 3.9 release.(`Release Candidate <http://www.gluster.org/pipermail/maintainers/2016-September/001442.html>`__ is available if anybody interested in testing)

Without Events APIs, one way to get status of Cluster is by calling Gluster status command/api in periodic intervals.

Below illustration shows calling status once every 10 seconds.

  1. image:: /images/gluster_monitor_without_events.jpg :alt: Get Cluster status without Events APIs

With Events APIs, setup a listener Webhook and register with Gluster using gluster-eventsapi webhook-add <URL>. Call Gluster status command whenever webhook receives an Event.(Check `this <http://aravindavk.in/blog/10-mins-intro-to-gluster-eventing/>`__ blog to know Webhooks and Events APIs in detail)

  1. image:: /images/gluster_monitor_with_events.jpg :alt: Get Cluster status with Events APIs

As part of presentation, created some visualizations to show how these real time notifications can be used to refresh the UI automatically when Gluster cluster state changes.
